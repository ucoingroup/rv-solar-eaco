## 第71章：私钥泄露后的应急操作

### 71.1 遇到最糟糕的情况

```
发现助记词或私钥可能泄露了
```

**不要慌，按秒行动。**

### 71.2 黄金 60 秒应急流程

```
0–10 秒：
  → 用另一个干净设备创建新钱包
  → 安装 Phantom 或 Backpack

10–30 秒：
  → 把旧钱包里所有资产转移到新钱包
  → 包括所有代币和 NFT

30–45 秒：
  → 检查是否有授权未撤销
  → 访问 https://revoke.cash
  → 撤销不需要的授权

45–60 秒：
  → 检查旧钱包是否还有隐藏资产（NFT、质押）
  → 确认转移完成
  → 废弃旧钱包
```

### 71.3 如果资产已经被转走

```
✅ 接受事实（追回概率极低）
✅ 在 Solscan 上查交易记录
✅ 报警（但几乎没用）
✅ 学习教训，重新开始
✅ 在社区提醒其他人
```

### 71.4 预防永远比补救重要

```
⭐⭐⭐⭐⭐ 助记词永不触网
⭐⭐⭐⭐⭐ 大额资产上硬件钱包
⭐⭐⭐⭐ 小额专用钱包交互
⭐⭐⭐ 定期检查授权
```

## 第72章：貔貅 / 蜜罐检测方法

### 72.1 什么是貔貅（Honeypot）？

```
你可以买入，但无法卖出

原理：
  → 合约代码中设置限制卖出的条件
  → 只有特定地址能卖出
  → 普通用户只能买，不能卖
```

### 72.2 如何检测？

| 工具 | 网址 | 检测能力 |
|------|------|---------|
| RugCheck.xyz | rugcheck.xyz | 自动检测貔貅 |
| Honeypot.is | honeypot.is | 专门检测卖出限制 |
| Token Checker | tokenchecker.app | 综合检测 |

### 72.3 手动检测信号

| 危险信号 | 说明 |
|----------|------|
| 只有"买入"交易，几乎没有"卖出" | 极可疑 |
| 持币地址分布极集中 | 前10地址 >50% |
| Liquidity 未锁仓 | 项目方可随时撤池 |
| 合约未开源 | 无法审计 |

### 72.4 EACO 的安全状态检查

| 检查项 | 建议 |
|--------|------|
| 合约是否开源 | 建议社区公布 |
| 流动性是否锁仓 | 请自行查询最新状态 |
| 持币分布 | 社区持续去中心化 |

> ⚠️ 每次交易前都要检查，不要因为"朋友推荐"而跳过。

## 第73章：授权盗币与 Revoke.cash

### 73.1 什么是授权盗币？

```
当你连接 DApp 并"批准"时，你可能给了它无限授权

无限授权 = 该 DApp 可以无限转走你的某种代币
```

**真实案例：**
```
某 DEX 被攻击
攻击者利用用户之前的无限授权
转走了数百万美元
```

### 73.2 Revoke.cash 使用教程

```
1. 访问 https://revoke.cash
2. 连接钱包
3. 选择 Solana 网络
4. 查看所有授权
5. 对不认识/不再使用的 DApp 点击 "Revoke"
```

### 73.3 建议检查频率

| 使用频率 | 建议检查周期 |
|----------|-------------|
| 重度 DeFi 用户 | 每周 1 次 |
| 普通用户 | 每月 1 次 |
| 偶尔交互 | 每季度 1 次 |

### 73.4 预防永远比补救重要

```
✅ 每次连接新 DApp 前，问自己：我真的信任它吗？
✅ 使用"限额授权"而不是"无限授权"
✅ 定期清理不需要的授权
```

## 第74章：钱包粉尘攻击与隐私保护

### 74.1 什么是粉尘攻击（Dusting Attack）？

```
你的钱包突然收到几毛钱的陌生代币

目的：
  → 追踪你的钱包活动
  → 关联你的其他钱包
  → 尝试钓鱼（代币附带恶意链接）
```

### 74.2 如何应对？

| 行为 | 正确 | 错误 |
|------|------|------|
| 看到小额陌生代币 | 忽略 | ❌ 点击链接 |
| 想"清除"它 | 不要交互 | ❌ 试图卖出/转移 |
| 想找到来源 | 用 Solscan 查 | ❌ 打开附带网址 |

### 74.3 隐私保护进阶

| 方法 | 效果 | 难度 |
|------|------|------|
| 不使用同一钱包跨多个平台 | 中 | ⭐ |
| 使用新钱包进行敏感操作 | 中 | ⭐ |
| Tornado Cash（不推荐）| 高（但违法）| ⭐⭐⭐ |

> ⚠️ 注意：隐私 ≠ 匿名。链上所有交易永远公开。

```
私钥永不离开设备
```


## 第79章：MEV 与三明治攻击

### 79.1 什么是 MEV？

**MEV = Maximal Extractable Value（最大可提取价值）**

```
有人利用"交易排序的权利"赚钱

你的交易被"夹"在两笔交易之间
导致你买贵了或卖便宜了
```

### 79.2 三明治攻击

```
攻击者发现你的买单
  ↓
在你之前买（抬高价格）
  ↓
你的买单成交（高价）
  ↓
攻击者卖出（获利）
```

### 79.3 Solana 上的 MEV

| 公链 | MEV 程度 | 原因 |
|------|---------|------|
| 以太坊 | 极高 | 内存池公开 + Gas 竞价 |
| Solana | 较低 | 无公开内存池 |

### 79.4 如何减少 MEV 风险？

| 方法 | 效果 |
|------|------|
| 使用 Jupiter（路由优化）| 中 |
| 不提交极高滑点 | 中 |
| 使用私有交易（Jito）| 高 |
| 小额交易 | 攻击者不感兴趣 |
