## 第16章：助记词抄写、备份与防丢方案

### 16.1 助记词是你的一切

**核心原则：**

> **丢失助记词 = 永久丢失钱包里所有资产**

这不是夸张，而是 Web3 的基本现实。

**对比：**

| 情况 | Web2（微信/支付宝）| Web3（钱包）|
|------|-------------------|-------------|
| 忘记密码 | ✅ 可以找客服重置 | ❌ 不可能 |
| 手机丢失 | ✅ 可以在新手机登录 | ✅ 可以用助记词恢复 |
| 助记词丢失 | 无此概念 | ❌ 永久无法恢复 |
| 账号被盗 | ✅ 客服可以帮你找回 | ❌ 没有客服 |

**这意味着：**

```
你的助记词 = 你的资产

保护好助记词 = 保护好资产
丢失助记词 = 丢失资产
泄露助记词 = 资产被盗
```

### 16.2 错误备份方式（永远不要用）

**方式一：截图保存**

```
为什么危险：
  → 黑客可以远程读取你的相册
  → 恶意 App 可能扫描相册
  → 云同步可能泄露

案例：
  → 某用户截图保存助记词
  → 手机中了木马
  → 黑客读取相册，找到助记词
  → 钱包资产被盗
```

**方式二：存微信/Telegram**

```
为什么危险：
  → 消息存储在平台服务器
  → 平台员工理论上可以查看
  → 账号被盗时，历史消息泄露

案例：
  → 某用户把助记词发给自己（文件传输助手）
  → 微信账号被盗
  → 骗子查看历史消息，找到助记词
  → 钱包资产被盗
```

**方式三：存云盘/备忘录**

```
为什么危险：
  → 云盘可能被黑
  → 备忘录同步到云端
  → 账号泄露时，所有数据泄露

案例：
  → 某用户把助记词存 iCloud 备忘录
  → iCloud 账号被盗
  → 骗子查看备忘录，找到助记词
  → 钱包资产被盗
```

**方式四：发给"信任的朋友"**

```
为什么危险：
  → 朋友也可能被黑
  → 朋友可能无意泄露
  → 关系变化时，信任基础变化

案例：
  → 某用户把助记词发给朋友保管
  → 朋友手机被黑
  → 黑客找到助记词
  → 钱包资产被盗
```

**方式五：存电脑文件**

```
为什么危险：
  → 电脑可能中病毒
  → 硬盘可能被读取
  → 维修时数据泄露

案例：
  → 某用户把助记词存 txt 文件
  → 电脑中了勒索病毒
  → 文件被加密并上传
  → 钱包资产被盗
```

**错误方式总结：**

| ❌ 方式 | 风险来源 |
|--------|---------|
| 截图保存 | 黑客、恶意 App、云同步 |
| 存微信/Telegram | 平台服务器、账号被盗 |
| 存云盘/备忘录 | 云端泄露、账号被盗 |
| 发给朋友 | 朋友被黑、关系变化 |
| 存电脑文件 | 病毒、维修泄露 |

### 16.3 正确备份方式详解

**基础备份（资产 $0–$1,000）**

**方案一：纸笔手写**

```
材料：
  → 一张干净的纸
  → 一支笔（不要用铅笔，会褪色）

步骤：
  1. 在安静环境下操作
  2. 按顺序写下 12 个单词
  3. 检查拼写和顺序
  4. 标记是哪个钱包的（如"Phantom 主钱包"）
  5. 放在安全的地方

存放建议：
  → 至少 2 份，不同地方
  → 例如：家里 1 份，父母家 1 份
  → 防火、防水、防盗

成本：$0
安全性：中（纸质易损坏）
```

**方案二：钢板刻字**

```
材料：
  → 专用助记词钢板（淘宝/亚马逊购买）
  → 或 DIY：不锈钢板 + 刻字笔

步骤：
  1. 在钢板上刻下助记词
  2. 按顺序刻
  3. 确认清晰可读
  4. 放在安全地方

优势：
  → 防火（钢铁熔点 1500°C）
  → 防水
  → 防腐蚀
  → 寿命长

成本：$20-50
安全性：高
```

**进阶备份（资产 $1,000–$10,000）**

**推荐使用金属助记词板：**

| 产品 | 价格 | 特点 |
|------|------|------|

**高级备份（资产 >$10,000）**

**方案一：多签名钱包**

```
什么是多签？
  → 需要多个私钥共同签名才能转账
  → 例如 2/3 多签：3 个人，任意 2 人同意即可

优势：
  → 单个私钥泄露不会丢资产
  → 需要多人共谋才能盗币

适合：
  → 家庭共同资产
  → 团队资金
  → DAO 国库
```

**方案二：Shamir Backup**

```
什么是 Shamir？
  → 把助记词拆分成多片
  → 需要多片才能恢复
  → 例如：拆成 5 片，任意 3 片可恢复

优势：
  → 单片泄露不会丢资产
  → 需要多片才能恢复

工具：
  → Ledger 支持 Shamir 备份
```

### 16.4 防丢检查清单

**创建备份后，检查以下项目：**

```
□ 助记词不在任何联网设备上
   → 没有截图
   → 没有存云盘
   → 没有发微信

□ 至少有 2 个物理备份

□ 备份不在同一个房间

   → 盗窃风险

□ 你记得备份放在哪里
   → 不是"藏得太好自己找不到"

□ 备份标记清楚

   → 创建时间

□ 你知道如何使用助记词恢复钱包
   → 在其他设备导入
   → 熟悉流程
```

### 16.6 助记词管理最佳实践

**原则一：永不触网**

```
助记词永远只在：
  → 纸上
  → 金属板上
  → 你的脑子里（如果记忆力极好）

助记词永远不在：
  → 手机
  → 电脑
  → 云端
  → 任何联网设备
```

**原则二：多份备份**

```
建议配置：
  → 2-3 份物理备份
  → 不同地点
  → 不同形式（纸 + 金属）

不要：
  → 只有一份（火灾/盗窃全丢）
  → 都在同一个地方
```

**原则三：定期检查**

```
建议频率：
  → 每 6 个月检查一次备份
  → 确认备份完好
  → 确认你记得存放位置
```

**原则四：告知信任的人**

```
如果你有大量资产：
  → 告诉至少一个信任的人备份位置
  → 写一份"数字遗产"指南
  → 防止意外时资产永久丢失
```

### 16.7 常见问题

**问题一：助记词顺序错了怎么办？**

```
如果你记不清顺序：
  → 尝试所有排列组合（12! = 479,001,600 种）
  → 实际上，可以用工具帮助恢复
  → 但很麻烦，所以一定要抄写时确认顺序

```

```
  → 钱包地址会完全不同
  → 资产无法找回

  → 抄写时仔细核对
  → 使用助记词验证工具
```

**问题三：我忘记助记词放哪了怎么办？**

```
如果资产不多：
  → 当作学费，创建新钱包
  → 以后记得备份位置

如果资产很多：
  → 仔细回忆
  → 翻遍可能的地方
  → 实在找不到，接受损失
```

### 17.4 授权盗币详解

**什么是授权盗币？**

```
当你连接 DApp 并"批准"时：
  → 你给了 DApp 一定权限
  → DApp 可以操作你的代币

如果你给了"无限授权"：
  → DApp 可以转走你所有该代币
  → 即使你现在不操作
  → 未来 DApp 被攻击时，攻击者可以利用你的授权
```

**案例：**

```
场景：
  1. 你在某 DEX 授权了 USDC
  2. 你选择了"无限授权"（默认选项）
  3. 半年后，该 DEX 被攻击
  4. 攻击者利用用户的授权，转走所有 USDC
  5. 你的 USDC 被盗

原因：
  → 你给了无限授权
  → 授权一直在链上
  → 攻击者可以利用
```

**如何防范？**

```
方法一：给限额授权
  → 只授权需要的金额
  → 不要选"无限"

方法二：定期清理授权
  → 访问 https://revoke.cash
  → 连接钱包
  → 查看所有授权
  → 撤销不需要的授权

方法三：使用专用钱包
  → 交互用小额钱包
  → 储蓄用单独钱包
  → 授权只影响小额钱包
```

## 第18章：DID 去中心化身份入门

### 18.1 什么是 DID？

**DID = Decentralized Identifier（去中心化身份）**

```
简单理解：
  → 你的钱包地址 = 你的链上身份证
  → 不依赖任何平台
  → 无法被封禁
```

### 18.2 Web2 身份 vs Web3 DID

**Web2 身份的问题：**

```
场景一：微信被封
  → 你的微信账号被封
  → 你的联系人丢失
  → 你的聊天记录丢失
  → 你的小程序数据丢失
  → 你需要重新建立一切

场景二：Google 账号被封
  → 你的 Gmail 无法使用
  → 你的 YouTube 频道丢失
  → 你的 Google Drive 数据丢失
  → 所有依赖 Google 登录的服务无法使用

核心问题：
  → 你的身份归平台所有
  → 平台可以随时剥夺
  → 你没有任何保障
```

**Web3 DID 的优势：**

| 维度 | Web2 身份 | Web3 DID |
|------|-----------|----------|
| 控制者 | 平台（Google/微信）| 你自己 |
| 所有权 | 平台拥有 | 你拥有 |
| 可封禁性 | 可以随时封禁 | 无法被封禁 |
| 跨平台使用 | 不能 | 可以 |
| 数据透明度 | 平台黑箱 | 公开可查 |
| 可移植性 | 低 | 高 |

### 18.3 DID 能做什么？

**应用一：登录 Web3 应用**

```
传统登录：
  → 注册账号
  → 设置密码
  → 绑定邮箱/手机
```

## 第19章：ENS / .sol 域名与链上身份

### 19.1 为什么需要域名？

**钱包地址的问题：**

```

7nVGH8xqLkM9pQt3YmN5hN4kR7vX2cT6sB1dE4fG

问题：
  → 太长（32-44 字符）
  → 难记
  → 容易复制错
  → 不专业（在简历/社交媒体）
```

**域名的解决方案：**

```
域名示例：john.sol

优势：
  → 简短好记
  → 不易出错
  → 专业（像邮箱）
  → 个人品牌
```

### 19.2 ENS（以太坊域名服务）

**ENS 是什么？**

```
ENS = Ethereum Name Service
→ 以太坊上的域名系统
→ 最成熟的区块链域名服务

域名格式：name.eth

示例：
  → vitalik.eth（Vitalik 的地址）
  → ethereum.eth（以太坊基金会）
```

**ENS 的功能：**

| 功能 | 说明 |
|------|------|
| 地址解析 | john.eth → 钱包地址 |
| 登录 Web3 | 用域名登录 |
| 个人品牌 | 在社交媒体展示 |
| 多链支持 | 可以解析多条链地址 |

**ENS 年费：**

| 域名长度 | 年费 |
|---------|------|
| 3 字符 | ~$640/年 |
| 4 字符 | ~$160/年 |
| 5+ 字符 | ~$5/年 |

### 19.3 .sol 域名（Solana）

**.sol 是什么？**

```
.sol = Solana 上的域名服务
→ 由 Bonfida 运营
→ 类似 ENS，但针对 Solana

域名格式：name.sol

示例：
  → john.sol
  → eaco.sol
```

**.sol vs ENS：**

| 维度 | ENS | .sol |
|------|-----|------|
| 公链 | Ethereum | Solana |
| 域名格式 | name.eth | name.sol |
| 年费 | $5-640 | ~$20 |
| 成熟度 | 最高 | 高 |
| 费用 | ETH | SOL/USDC |
