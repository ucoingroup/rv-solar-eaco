## 第21章：SPL / ERC-20 代币标准详解

### 21.1 什么是代币标准？

**代币标准 = 一套规则，规定了代币如何发行、转账、授权**

类比理解：

| 现实世界 | Web3 |
|----------|------|
| USB标准 | 代币标准 |
| 规定接口形状、数据传输方式 | 规定代币发行、转账、授权方式 |
| 任何USB设备都能插入任何USB接口 | 任何符合标准的代币都能被钱包识别 |

**为什么需要代币标准？**

```
没有标准：
  → 每个项目自己定义代币格式
  → 钱包需要适配每个项目
  → DEX需要适配每个项目
  → 生态混乱

有标准：
  → 所有代币遵循统一格式
  → 钱包只需适配标准
  → DEX只需适配标准
  → 生态统一
```

### 21.2 Solana SPL Token 标准详解

**SPL = Solana Program Library**

SPL Token 是 Solana 上的代币标准，由 Solana 官方维护。

**SPL Token 的特点：**

| 特点 | 说明 |
|------|------|
| ATA机制 | Associated Token Account，每个代币有独立账户 |
| 精度 | 可配置，通常 6-9 位小数 |
| 铸造/销毁 | 支持增发和销毁 |
| 授权 | 支持委托授权 |
| Gas | 极低（$0.001左右）|

**ATA（Associated Token Account）机制详解：**

```
以太坊 ERC-20：
  → 一个地址持有所有代币
  → 代币余额记录在代币合约中

Solana SPL：
  → 每个代币有独立的代币账户
  → 你的钱包地址 = 主账户
  → 你持有的 USDC = USDC 代币账户
  → 你持有的 EACO = EACO 代币账户

示例：
  主账户：7nVGH8xqLkM9pQt3YmN5hN4kR7vX2cT6sB1dE4fG
  USDC账户：自动生成的ATA地址
  EACO账户：自动生成的ATA地址
```

**SPL Token 的优势：**

```
优势一：Gas极低
  → Solana 交易费约 $0.001
  → 以太坊可能 $5-50
  → 适合高频交易

优势二：速度快
  → 400ms 出块
  → 几秒确认
  → 适合实时交互

优势三：账户模型
  → 每个代币独立账户
  → 并行处理
  → 高吞吐量
```

### 21.3 Ethereum ERC-20 标准详解

**ERC-20 = Ethereum Request for Comments 20**

ERC-20 是以太坊上最广泛使用的代币标准。

**ERC-20 核心函数：**

```solidity
// 查询余额
function balanceOf(address account) external view returns (uint256);

// 转账
function transfer(address to, uint256 amount) external returns (bool);

// 查询授权额度
function allowance(address owner, address spender) external view returns (uint256);

// 授权他人使用你的代币
function approve(address spender, uint256 amount) external returns (bool);

// 授权转账（被授权者调用）
function transferFrom(address from, address to, uint256 amount) external returns (bool);
```

**approve 授权机制详解：**

```
场景：你在 Uniswap 用 USDC 买 ETH

步骤：
  1. 你调用 USDC.approve(uniswap, 1000)
     → 授权 Uniswap 使用你 1000 USDC

  2. 你调用 Uniswap.swap()
     → Uniswap 调用 USDC.transferFrom(you, uniswap, 1000)
     → 把你的 USDC 转给 Uniswap

风险：
  → 如果你授权了无限额度
  → Uniswap 被攻击后，攻击者可以转走你所有 USDC
  → 所以建议只授权需要的额度
```

### 21.4 SPL vs ERC-20 全面对比

| 维度 | SPL (Solana) | ERC-20 (Ethereum) |
|------|--------------|-------------------|
| 架构 | 账户模型 | 合约存储 |
| 代币账户 | 每个代币独立账户 | 统一在合约中记录 |
| Gas费用 | 极低（$0.001）| 高（$5-50）|
| 交易速度 | 快（400ms）| 慢（12s）|
| 授权机制 | 委托授权 | approve/allowance |
| 并行处理 | 支持 | 不支持 |
| 开发难度 | 中等 | 低（生态成熟）|
| 生态规模 | 中等 | 最大 |

### 21.5 跨链代币（Wrapped Token）

**什么是跨链代币？**

```
Wrapped Token = 在一条链上代表另一条链资产

示例：
  wBTC = Wrapped Bitcoin
  → 在以太坊上代表 BTC
  → 1 wBTC = 1 BTC
  → 可以在以太坊 DeFi 中使用

  wETH = Wrapped ETH
  → 在 Solana 上代表 ETH
  → 通过跨链桥铸造
```

**跨链桥工作原理：**

```
步骤一：锁定
  → 你在以太坊锁定 1 ETH
  → 锁定合约保管你的 ETH

步骤二：铸造
  → 跨链桥在 Solana 铸造 1 wETH
  → 发送到你的 Solana 地址

步骤三：使用
  → 你在 Solana 上使用 wETH
  → 可以交易、提供流动性

步骤四：赎回
  → 你销毁 Solana 上的 wETH
  → 跨链桥释放以太坊上的 ETH
  → 你收到 1 ETH
```

### 21.7 授权风险与限额授权

## 第22章：稳定币：USDC / USDT / DAI 如何选择

### 22.1 什么是稳定币？

**稳定币 = 锚定法币价值的代币**

```
稳定币的定义：
  → 1 USDC ≈ 1 美元
  → 1 USDT ≈ 1 美元
  → 1 DAI ≈ 1 美元

为什么需要稳定币：
  → 加密货币波动大（BTC一天可能跌20%）
  → 需要稳定的价值储存
  → 需要稳定的交易媒介
  → DeFi 借贷需要稳定计价
```

**稳定币的分类：**

| 类型 | 代表 | 抵押方式 | 去中心化程度 |
|------|------|---------|-------------|
| 法币抵押 | USDC、USDT | 银行存款 | 中心化 |

| 算法稳定币 | UST（已崩盘）| 无抵押 | 去中心化 |

  → USDC：Circle 破产风险
  → USDT：Tether 储备不足风险

  → 不要长期大量持有稳定币

### 25.4 无常损失（Impermanent Loss）详解

**什么是无常损失？**

```
无常损失 = 提供流动性 vs 单纯持有的收益差异

示例：
  初始：
    → EACO 价格 $0.001
    → 你提供 1000 EACO + 1 USDC
    → 总价值 $2

  后来：
    → EACO 价格涨到 $0.002
    → 池中比例自动调整
    → 你持有的 EACO 减少，USDC 增加
    → 你的 LP 价值约 $2.8

  对比：
    → 如果单纯持有：1000 EACO = $2，1 USDC = $1，共 $3
    → 提供 LP：$2.8
    → 无常损失 = $3 - $2.8 = $0.2（约 7%）

关键：
  → 价格变化越大，无常损失越大
  → 手续费收益可能弥补无常损失
```

**无常损失计算公式：**

```
IL = 2 * sqrt(price_ratio) / (1 + price_ratio) - 1

示例：
  价格翻倍（price_ratio = 2）：
    IL = 2 * sqrt(2) / (1 + 2) - 1
    IL = 2 * 1.414 / 3 - 1
    IL = -5.7%

  价格涨 4 倍（price_ratio = 4）：
    IL = 2 * sqrt(4) / (1 + 4) - 1
    IL = 2 * 2 / 5 - 1
    IL = -20%
```

**无常损失表格：**

| 价格变化 | 无常损失 |
|---------|---------|
| 1.25x（涨25%）| -0.6% |
| 1.5x（涨50%）| -2.0% |
| 2x（翻倍）| -5.7% |
| 3x（涨3倍）| -13.4% |
| 4x（涨4倍）| -20.0% |
| 5x（涨5倍）| -25.5% |
