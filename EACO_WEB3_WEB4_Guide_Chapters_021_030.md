## 第21章：SPL / ERC-20 代币标准详解

### 21.1 什么是代币标准？

**代币标准 = 一套规则，规定了代币如何发行、转账、授权**

就像 USB 标准规定了接口形状和数据传输方式，代币标准规定了：

- 代币如何创建
- 代币如何转账
- 代币如何授权他人使用

### 21.2 主要公链的代币标准

| 公链 | 代币标准 | 代表代币 | Gas 费 |
|------|----------|----------|--------|
| Solana | SPL | EACO、USDC、SOL（原生）| 极低（$0.001）|
| Ethereum | ERC-20 | USDT、UNI、LINK | 高（$5–50）|
| BNB Chain | BEP-20 | CAKE、BNB | 低（$0.1）|
| Polygon | ERC-20 兼容 | MATIC | 中（$0.01）|

### 21.3 SPL 代币（SPL = Solana Program Library）

SPL 是 Solana 上的代币标准，有几个独特特点：

**特点一：关联代币账户（ATA）**

```
传统理解（错误）：代币直接存在钱包里
实际情况：正确理解如下

当你持有 SPL 代币时：
钱包地址
  └── ATA（关联代币账户）
        └── 代币余额

Phantom 会自动为你创建 ATA
你不需要手动操作
```

**特点二：不需要 Gas 费（有时）**

- 转账 SPL 代币时，通常不需要额外 SOL 作为 Gas
- 但创建新 ATA 时可能需要少量 SOL

### 21.4 ERC-20 代币（以太坊）

ERC-20 是最成熟的代币标准：

**核心函数：**

```solidity
function transfer(address to, uint256 amount)  // 转账
function approve(address spender, uint256 amount) // 授权
function transferFrom(address from, address to, uint256 amount) // 从他人账户转
```

**授权机制详解：**

```
场景：你想让 DEX 可以帮你交易代币
步骤：
1. 你调用 approve(dexAddress, 1000)
   → 允许 DEX 最多转走 1000 代币
2. DEX 调用 transferFrom(你的地址, 对方地址, 500)
   → DEX 转走 500 代币
3. 你还剩 500 授权额度
```

⚠️ **风险**：如果授权了无限额度，而 DEX 被攻击，你可能损失所有代币。

### 21.6 跨链代币

同一个名字的代币，在不同链上是不同的代币：

| 代币 | Solana 版本 | Ethereum 版本 | 关系 |
|------|-----------|-------------|------|
| USDC | SPL（原生快速）| ERC-20 | 1:1 锚定 |
| USDT | SPL（需要跨链桥）| ERC-20（原生）| 1:1 锚定 |
| WBTC | Wormhole WBTC | ERC-20 WBTC | 1:1 锚定 |

**跨链桥的作用：**

```
你在 Solana 上有 USDC（SPL）
想转到 Ethereum 上

跨链桥操作：
1. 在 Solana 锁定 100 USDC
2. 跨链桥在 Ethereum 铸造 100 USDC（ERC-20）给你

跨链 USDC 的价值仍然是 1:1
```

## 第22章：稳定币：USDC / USDT / DAI 如何选择

### 22.1 什么是稳定币？

**稳定币 = 价格锚定法币的加密资产**

1 USDC ≈ 1 美元（始终锚定）

### 22.2 三大稳定币对比

| 属性 | USDC | USDT | DAI |
|------|------|------|-----|
| 发行方 | Circle | Tether | MakerDAO |
| 锚定 | 1 USD | 1 USD | 1 USD |
| 机制 | 法币抵押 | 法币抵押 | 加密资产超额抵押 |
| 透明度 | 高（月度审计）| 中（争议较大）| 高（链上公开）|
| 中心化程度 | 高度中心化 | 高度中心化 | 去中心化 |
| 冻结风险 | 可冻结 | 可冻结 | 不可冻结 |
| Solana 支持 | ✅ 原生 | ✅ 跨链桥 | ⚠️ 较少 |
| 发行时间 | 2018 | 2014 | 2017 |

### 22.3 为什么 Solana 上推荐 USDC？

**USDC 的优势：**

| 优势 | 说明 |
|------|------|
| Circle 合规 | 美国监管最严格的稳定币发行方 |
| Solana 原生 | 无需跨链桥，直接原生支持 |
| 交易费极低 | < $0.01 / 笔 |
| 生态支持 | 所有 Solana DEX/DeFi 都支持 |

**USDT 在 Solana 上的问题：**

- 需要通过 Wormhole 等跨链桥跨过来
- 跨链过程有风险
- 如果跨链桥出问题，USDT 可能无法转回

### 22.4 DAI 的去中心化优势

**为什么选择 DAI：**

```
USDC/USDT：中心化风险
  → 发行方可以冻结你的资产
  → 如果发行方破产，储备不足

DAI：去中心化优势
  → 没有人可以冻结你的 DAI
  → 抵押品在链上透明
  → 更符合 Web3 的精神
```

**DAI 的劣势：**

- Solana 上支持较少
- 抵押机制复杂，可能有清算风险
- 存款利率通常比 USDC 低

### 22.5 EACO 与稳定币的关系

**在 Jupiter 上交易 EACO 时：**

| 操作 | 使用稳定币 |
|------|-----------|
| 买入 EACO | USDC → EACO |
| 卖出 EACO | EACO → USDC |
| 存储价值 | USDC（稳定，等同美元）|

**建议策略：**

```
持有 EACO 之外，始终保持一定比例的 USDC：
- 用于买入更多 EACO（趁低吸纳）
- 用于支付 Gas（SOL）
- 用于日常生活消费（通过加密卡）
```

## 第26章：滑点、优先级费用与交易失败处理

### 26.1 滑点详解

**什么是滑点？**

```
预期成交价 vs 实际成交价 的差异

场景：你用 Jupiter 买 EACO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
你看到的 EACO 价格：$0.001
你下单：买入 1000 EACO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
在你等待确认时：
  别人抢先买了 500 EACO
  EACO 价格变成 $0.0011
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
你实际成交价：$0.0011
滑点：10%（价格上涨了 10%）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**滑点的来源：**

1. 区块链确认需要时间（SOL 约 0.4 秒一个区块）
2. 你的交易在等待时，价格可能变动
3. 特别是流动性低的池子，价格变动更大

### 26.2 滑点设置建议

| 代币类型 | 推荐滑点 | 备注 |
|----------|----------|------|
| SOL/USDC | 0.5–1% | 流动性极好 |
| BTC/ETH | 0.5–1% | 流动性好 |
| 主流 DeFi 代币 | 1–2% | 流动性较好 |
| EACO（稳定期）| 1–3% | Meme 代币，波动适中 |

|----------|------|----------|
| 滑点过低 | "交易失败：价格变动太大" | 提高滑点 |

| 网络拥堵 | 等待时间长后失败 | 增加优先级费用 |
| 代币账户未创建 | 罕见 | Phantom 通常自动处理 |
| 钱包拒绝 | 签名被拒绝 | 重新确认签名 |

**滑点失败的正确应对：**

```
如果提示 "Slippage exceeded"
1. 不要重复提交相同交易（价格可能继续变）
2. 提高滑点（从 1% → 3%）
3. 等待一会儿再试
4. 如果还是不行，考虑降低金额
```

## 第27章：链上流动性原理（AMM）

### 27.1 什么是 AMM？

**AMM = Automated Market Maker（自动化做市商）**

**传统订单簿 vs AMM：**

| 传统订单簿 | AMM |
|-----------|-----|
| 买卖双方挂单 | 流动性池自动定价 |
| 需要撮合才能成交 | 随时可交易 |
| 大户定价 | 数学公式定价 |
| 典型：纳斯达克 | 典型：Uniswap、Raydium |

### 27.2 AMM 的定价公式

**最简单的 AMM：恒定乘积公式**

```
x × y = k

x = 代币 A 的数量
y = 代币 B 的数量
k = 恒定值（不变）

━━━━━━━━━━━━━━━━━━━━━━━━━━
示例：EACO/USDC 池

初始状态：
  EACO = 1,000,000（100万）
  USDC = 1,000（$1000）
  k = 1,000,000 × 1,000 = 1,000,000,000

━━━━━━━━━━━━━━━━━━━━━━━━━━
有人买入 100,000 EACO：

新状态：
  EACO = 1,100,000
  USDC = 1,000,000,000 / 1,100,000 ≈ 909.09
  买家付出：1,000 - 909.09 = 90.91 USDC
  每 EACO 价格 = 90.91 / 100,000 ≈ $0.00091

━━━━━━━━━━━━━━━━━━━━━━━━━━
EACO 越买越贵（池子变深后趋于平稳）
```

### 27.3 无常损失（Impermanent Loss）

**无常损失 = LP 收益 vs 单纯持有 的差异**

```
场景：你有 1 SOL + 100 USDC（总计 $200，假设 1 SOL = $100）

方案 A：单纯持有
  → 1 SOL + 100 USDC = $200

方案 B：添加 LP（50% SOL + 50% USDC）
  → 如果 SOL 涨到 $200：
    LP 里变成 0.5 SOL + 100 USDC = $200
    单纯持有 = 1 SOL = $200
    你少了什么？机会成本 + 手续费补偿

━━━━━━━━━━━━━━━━━━━━━━━━━━
无常损失不是"损失"，而是"相对机会"
只有在取出时才会"实现"
```

**什么时候适合做 LP？**

```
✅ 适合做 LP：
  - 你同时看好两个代币
  - 两个代币波动方向相反（稳定币对）
  - 你需要赚取交易手续费
  - 你愿意接受无常损失风险

❌ 不适合做 LP：
  - 单边行情（一个代币暴涨）
  - 你持有大量 EACO（流动性不足时）
  - 不理解无常损失
```

## 第29章：代币解锁、锁仓与通胀模型

### 29.1 代币解锁（Token Unlock）

**为什么代币要分批解锁？**

```
避免：大量代币一次性涌入市场 → 价格暴跌
方案：团队/VC/社区代币分批释放

━━━━━━━━━━━━━━━━━━━━━━━━
示例：某代币总供应量 10 亿
━━━━━━━━━━━━━━━━━━━━━━━━
解锁时间表：
  团队解锁：每月解锁 1%（1年后）
  VC 解锁：每月解锁 2%（6个月后）
  社区激励：每天解锁 0.1%
━━━━━━━━━━━━━━━━━━━━━━━━
如果一次性全部解锁：
  市场会收到全部代币 → 抛压极大
```

### 29.2 锁仓（Vesting）

**什么是锁仓？**

```
锁仓 = 代币被锁定一段时间，不能交易

类型：
  - 团队锁仓：团队代币锁定 1-2 年
  - VC 锁仓：投资人代币有解锁期
  - 顾问锁仓：顾问代币通常 1 年
```

**如何查询代币解锁？**

| 工具 | 网址 |
|------|------|
| TokenUnlocks | tokenunlocks.com |
| Solscan | solscan.io |
| Dune Analytics | dune.com |

### 29.3 Solana 的通胀模型

**SOL 的通胀率：**

| 时间 | 年化通胀率 |
|------|-----------|
| 当前 | ~4–5% |
| 长期目标 | ~1.5% |
| 通胀去向 | 质押奖励 |

**SOL 通胀 = 稀释吗？**

```
不完全是。

SOL 通胀的目的：
  → 奖励验证者和质押者
  → 维护网络安全

如果你的 SOL 质押了：
  通胀 ≈ 你的质押收益
  
如果你的 SOL 没有质押：
  通胀 = 你的稀释

⚠️ 持有 SOL 不质押 ≈ 被动稀释
```
